## 3. Major clarifications (M1–M4)

### 3.1 M1 — Bootstrap re-estimate (α̂, β̂) per iteration (a-Major-1)

**Reviewer concern**: v2.0 Section 5.2 estimates the Beta-Binomial hyperprior (α̂, β̂) by method-of-moments from the 14-cell point estimates {p̂_j}, then uses (α̂, β̂) as fixed prior for 28-cell shrinkage. Because each 14-cell estimate p̂_j is a deterministic function of the 28-cell counts whose posteriors are being constructed, the standard EB independence assumption (prior and data are independent draws from the hyper-population) is violated. Reported posterior CIs from Beta(α̂+X_k, β̂+N_k−X_k) are therefore narrower than honest credibility would warrant.

**Clarification (Level 1, applied to v2.0 master Section 5.2)**:

The bootstrap procedure used to construct CIs around the 28-cell EB-shrunken estimates re-estimates the hyperprior (α̂^(b), β̂^(b)) on each bootstrap iteration b ∈ {1..2,000}. The full per-iteration procedure is:

1. Resample N=354 with replacement, stratified by 14 cell (per Section 5.4 cell-stratified spec).
2. Re-classify each resampled individual to the nearest of 7 centroids (centroids fixed from N=13,668; per M3 below).
3. Compute the 14-cell observed propensities {p̂_j^(b)} for j = 1..14 on the resampled data.
4. **Re-estimate the hyperprior on iteration b**:
   - μ̂^(b) = mean(p̂_j^(b)), σ̂²^(b) = var(p̂_j^(b))
   - α̂^(b) = μ̂^(b) × [μ̂^(b)(1−μ̂^(b))/σ̂²^(b) − 1]
   - β̂^(b) = (1−μ̂^(b)) × [μ̂^(b)(1−μ̂^(b))/σ̂²^(b) − 1]
5. Apply the hyperprior (α̂^(b), β̂^(b)) to compute the 28-cell posterior on iteration b: E[p_k | X_k^(b), N_k^(b)] = (α̂^(b) + X_k^(b)) / (α̂^(b) + β̂^(b) + N_k^(b)).
6. Record the 28-cell posterior values for iteration b.

After all B = 2,000 iterations, 95% bootstrap CIs for each 28-cell posterior are constructed from the empirical distribution of {posterior^(b)}, with BCa correction (per Section 5.1 fallback chain if numerical instability detected).

**Rationale**: This adopts the reviewer's option (i) — "naive plug-in EB with bootstrap propagation" (Carlin & Louis 2000; Efron 2014) — as the operational choice. Option (ii) (full hierarchical Bayes via Stan as primary 28-cell estimator) remains available as the auxiliary triangulation per Section 5.2; the present clarification ensures the EB CIs reported in the main text honestly propagate hyperprior uncertainty.

**Implementation impact**: Stage 0 code module `code/stage0_eb_shrinkage.py` must re-estimate (α̂^(b), β̂^(b)) inside the bootstrap loop, not outside it. Computational cost is negligible (the MoM formula is O(K) for K=14 cells, dwarfed by the overall bootstrap cost).

### 3.2 M2 — Soft-assignment sensitivity sweep (a-Major-2)

**Reviewer concern**: v2.0 Stage 0 hard-assigns each N=354 individual to the nearest of 7 centroids by Euclidean distance, treating cluster membership as deterministic. Individuals near a centroid boundary (similar distances to ≥ 2 centroids) have their assignment fixed across all bootstrap iterations, systematically underestimating cell-level CIs by omitting cluster-assignment uncertainty.

**Clarification (Level 1, applied to v2.0 master Section 6.4 sensitivity sweep table)**:

A new soft-assignment sensitivity is added to the pre-registered sweep:

| Parameter | Main | Sweep range | Stage |
|---|---|---|---|
| **Cluster assignment scheme** | Hard nearest-neighbor (default, primary) | {Hard nearest-neighbor; Soft assignment with τ_softmax ∈ {0.5, 1.0, 2.0} × median nearest-neighbor distance} | 0 |

**Soft assignment specification**:

For each individual i and each cluster j ∈ {0..6}, define:

- d_ij = Euclidean distance from i's HEXACO 6-domain vector to centroid j
- d_i^(NN) = min_j d_ij (nearest-neighbor distance for individual i)
- median(d_i^(NN)) = median of d_i^(NN) across i = 1..354
- Temperature parameter τ = c × median(d_i^(NN)) for c ∈ {0.5, 1.0, 2.0}
- Soft-assignment weight: w_ij(τ) = exp(-d_ij / τ) / Σ_k exp(-d_ik / τ)

Under soft assignment, individual i contributes weight w_ij(τ) to cluster j's cell count rather than counting fully in one cluster. The 14-cell crosstab becomes:

X_c (soft, c = (j, gender)) = Σ_{i with gender = gender_c} w_ij(τ) × Y_i
N_c (soft, c) = Σ_{i with gender = gender_c} w_ij(τ)

where Y_i is the binary harassment outcome.

The full Stage 0 → Stage 1 → Stage 2 pipeline is run separately for each c ∈ {0.5, 1.0, 2.0} and reported alongside the hard-assignment main analysis.

**Robustness criterion**: if the national MAPE (Stage 2 primary) shifts by < 5 percentage points across c ∈ {0.5, 1.0, 2.0}, the hard nearest-neighbor assignment is reported as the primary specification with soft-assignment sensitivity supplied as supplementary. If MAPE shifts by ≥ 5 pp across c, the hard assignment is downgraded from "robust to assignment scheme" to "sensitive to assignment scheme", and the Discussion explicitly acknowledges that cluster-boundary individuals materially affect the inference.

**Implementation impact**: Stage 0 code module `code/stage0_type_assignment.py` must support both hard and soft assignment modes, parameterized by τ. Stage 0 → Stage 2 pipeline is run 4× total (1 hard + 3 soft τ values). The 4 runs share the same bootstrap seed (20260429) and centroid file.

### 3.4 M4 — Degenerate-cell handling (b-Major-4)

**Reviewer concern**: v2.0 Section 5.1 BCa fallback chain (BCa → BC → percentile) does not explicitly handle the fully degenerate case where a cell has all-zero (X_c = 0) or all-one (X_c = N_c) outcomes. The BCa acceleration parameter is undefined in this case (jackknife produces zero variance). In the 14-cell main analysis with min N = 10, this is improbable but possible. In the 28-cell EB sensitivity with cells of N = 1–3, it is virtually guaranteed for several cells. Silently producing NaN or [0, 0] CIs would be a Methods bug.

**Clarification (Level 1, applied to v2.0 master Sections 5.1 and 5.2)**:

A degeneracy rule is added at the top of the BCa fallback chain. The full per-cell CI selection logic, in priority order, is:

1. **Degeneracy check**:
   - If X_c = 0 (no perpetrators in cell): use the **exact Clopper-Pearson interval** for the binomial parameter with X = 0, N = N_c, confidence 95% [Clopper & Pearson 1934]. This yields CI = [0, 1 − (0.025)^(1/N_c)] for the upper bound only.
   - If X_c = N_c (all perpetrators in cell): use exact Clopper-Pearson with X = N_c, N = N_c. This yields CI = [(0.025)^(1/N_c), 1] for the lower bound only.
2. **BCa primary** (if not degenerate):
   - Compute jackknife acceleration parameter a per Efron (1987) Eq. 6.6 (per m4 below).
   - If |a| ≤ 10 and BCa computation succeeds without numerical error, use BCa interval.
3. **BC fallback** (if BCa fails):
   - If |a| > 10 OR BCa computation raises numerical error (e.g., division by zero at edge rates), use bias-corrected (BC) bootstrap.
4. **Percentile fallback** (if BC also fails):
   - Use simple percentile bootstrap (2.5th and 97.5th quantiles of the resample distribution).

For the 28-cell EB tier, degeneracy at the cell level is mitigated upstream by the EB shrinkage itself: the prior pulls the posterior off the {0, 1} boundary even when X_k = 0 or X_k = N_k. Therefore, the 28-cell tier uses the **EB posterior credible interval** (from Beta(α̂^(b) + X_k, β̂^(b) + N_k − X_k) quantiles, per M1) as the primary CI, with the bootstrap CI from the BCa/BC/percentile chain as auxiliary triangulation.

**Per-cell method recording**: every cell in the supplementary table records which CI method was actually used: {Clopper-Pearson, BCa, BC, percentile} for the 14-cell tier, and {EB posterior, bootstrap-BCa, bootstrap-BC, bootstrap-percentile} for the 28-cell tier. This enables readers to verify that the fallback chain's selections do not produce qualitatively different conclusions across cells.

**Rationale**: The reviewer correctly identified that the BCa fallback chain literally cannot run on degenerate cells (acceleration undefined). Clopper-Pearson is the exact-binomial fallback for the 14-cell case (X_c ∈ {0, N_c}); the EB posterior is the natural fallback for the 28-cell case where the prior provides regularization. Both are standard methods; no new statistical machinery is introduced.

**Implementation impact**: Stage 0 code module `code/utils_bootstrap.py` (or equivalent CI utility) must implement the 4-step priority chain. Standard libraries (`scipy.stats.binom.interval` for Clopper-Pearson; `scipy.stats.beta.interval` for the EB posterior) cover both fallbacks. Total additional implementation cost: ~2 hours.

## 4. Minor clarifications (m1–m7)

### 4.1 m1 — MoM threshold bootstrap stabilization (a-Minor-1)

**Reviewer concern**: The v2.0 MoM rejection rule "σ̂² / [μ̂(1−μ̂)] < 0.05 → switch to Stan" is a hard switch on σ̂², which is itself estimated from K = 14 cell estimates with non-trivial sampling variability. A single unlucky 14-cell sample could trigger Stan switch when the underlying truth is borderline.

**Clarification (Level 1, applied to v2.0 master Section 5.2)**:

The MoM rejection rule is bootstrap-stabilized:

1. Draw 200 bootstrap resamples of the 14-cell estimates (resampling cells with replacement at the cell level, NOT the individual level).
2. For each bootstrap iteration, recompute σ̂²^{(b)} / [μ̂^{(b)}(1−μ̂^{(b)})].
3. Compute the upper 95% bound (97.5th percentile) of this ratio across the 200 iterations.
4. **Reject MoM (switch to Stan) only if the upper 95% bound < 0.05**.

This transforms a single-point hard switch into a robust statistical decision: MoM is rejected only when even the optimistic upper-tail estimate of the variance ratio falls below the threshold.

**Implementation impact**: ~10 minutes additional code (200-iteration outer loop on existing 14-cell bootstrap). Computational cost negligible.

### 4.2 m2 — Third MoM rejection trigger: α̂ + β̂ vs median cell N (a-Minor-2)

**Reviewer concern**: The pseudo-count interpretation of α̂ + β̂ is the effective prior sample size. When α̂ + β̂ ≫ median cell N (say, > 5×), the prior dominates and shrinkage is severe — a different failure mode than the variance-collapse case caught by m1.

**Clarification (Level 1, applied to v2.0 master Section 5.2)**:

A third MoM rejection trigger is added in parallel to the existing two:

| Trigger | Condition | Rationale |
|---|---|---|
| Trigger 1 (existing) | σ̂² / [μ̂(1−μ̂)] < 0.05 (with bootstrap stabilization per m1) | Variance collapse → extreme α̂, β̂ |
| Trigger 2 (existing) | max(α̂, β̂) > 100 | Asymmetric overshrinkage in one direction |
| **Trigger 3 (new, m2)** | **α̂ + β̂ > 5 × median(N_k for k in 28 cells)** | **Symmetric overshrinkage: prior dominates 28-cell evidence** |

If ANY of the three triggers fires, MoM is rejected and Stan hierarchical Bayes is used as the primary 28-cell estimator. All three triggers are reported with their numerical values in supplementary, regardless of whether they fired.

**Rationale**: The three triggers cover three distinct failure modes (variance collapse, asymmetric overshrink, symmetric overshrink). Pre-specifying all three provides defense-in-depth against MoM artifacts.

**Implementation impact**: ~5 minutes (add one comparison to the diagnostic block).

### 4.4 m4 — BCa acceleration estimation method specified (b-Minor-4)

**Reviewer concern**: v2.0 doesn't specify whether the BCa acceleration parameter `a` is computed via (i) jackknife (Efron 1987 original), (ii) negative skewness of the influence function, or (iii) other methods. For continuous outcomes the methods agree closely; for binary outcomes at small N, they can differ.

**Clarification (Level 1, applied to v2.0 master Section 5.1)**:

The BCa acceleration parameter `a` is computed via **jackknife following Efron (1987) Eq. 6.6**:

$$ a = \frac{\sum_{i=1}^{n} (\hat{\theta}_{(\cdot)} - \hat{\theta}_{(i)})^3}{6 \left[ \sum_{i=1}^{n} (\hat{\theta}_{(\cdot)} - \hat{\theta}_{(i)})^2 \right]^{3/2}} $$

where:
- $\hat{\theta}_{(i)}$ is the statistic computed with observation i deleted (jackknife replicate)
- $\hat{\theta}_{(\cdot)} = (1/n) \sum_i \hat{\theta}_{(i)}$ is the jackknife mean

The same formula applies at the cell level (n = N_c, jackknife over individuals within cell c) and at the headline MAPE level (n = N = 354, jackknife over the full sample with cell-stratified preservation).

If the jackknife denominator is zero or near-zero (degenerate case caught by M4), the cell falls into the M4 fallback chain.

**Implementation impact**: Standard `scipy.stats.bootstrap` and `arch.bootstrap` libraries implement Efron (1987) jackknife BCa by default. Verify the library version and method in the supplementary.

### 4.5 m5 — Positivity quantitative criterion (c-Minor-5)

**Reviewer concern**: v2.0 Section 5.7.4 honestly flags positivity concerns for Counterfactual B (Cluster 6 dominance, role=1 empty cells), but the discussion stops at "do not exclude Cluster 6 from analysis." This addresses the wrong direction of positivity violation. The genuine concern is the OPPOSITE: for cells with very few or zero observations of the target Cluster (0/4/6) under treated conditions, the counterfactual involves extrapolation from cells where the treatment was effectively never observed. The structural causal model machinery doesn't fail (computation proceeds), but the result is essentially extrapolation.

**Clarification (Level 1, applied to v2.0 master Section 5.7.4 + Section 5.7.3)**:

A positivity diagnostic is added as a pre-registered criterion within Phase 2 counterfactual estimation:

**Positivity diagnostic (computed for each (cell, counterfactual) pair)**:

For each 14-cell c and each counterfactual x ∈ {A, B, C}:

1. Compute the **propensity score-equivalent ratio** ρ_{c,x}:
   - For Counterfactual A (universal): ρ_{c,A} = 1 (all cells receive treatment; no extrapolation)
   - For Counterfactual B (targeted on Clusters 0/4/6):
     ρ_{c,B} = (observed individuals in target Cluster within cell c) / (expected post-intervention count in cell c)
     where "expected post-intervention count" is the number of individuals projected to fall in the target cluster after the δ_B HEXACO shift.
   - For Counterfactual C (structural multiplier):
     ρ_{c,C} = 1 (cell-level multiplier, no extrapolation)

2. **Flag a cell as positivity-violated** if ρ_{c,x} < 0.10 for that counterfactual (i.e., the counterfactual involves > 10× extrapolation from observed evidence in that cell).

3. Compute the **proportion of total population weight that flows through flagged cells**:
   - flagged_weight_x = Σ_{c flagged} W_c / Σ_{c} W_c

4. **Tier downgrade rule** (pre-registered):
   - If flagged_weight_x < 20% of total: ΔP_x is reported with normal Phase 2 confirmatory framing.
   - If flagged_weight_x ≥ 20% of total: **ΔP_x is downgraded from confirmatory to exploratory**. The Phase 2 hypothesis (H4 for x=A, H5 for x=B, H6 for x=C) is reported as "exploratory under positivity violation; confirmatory inference deferred to future studies with broader cluster coverage."

**Application**: This rule applies to H4 (Counterfactual A), H5 (Counterfactual B), H6 (Counterfactual C). It does NOT affect H1 (Phase 1 main, no counterfactual involved) or H2 / H2.industry / H3 (Phase 1 baselines). H7 (Phase 2 main contrast) inherits the downgrade if any of its component ΔP_x is downgraded; the contrast is reported as exploratory if H4 / H5 / H6 are all downgraded, otherwise as confirmatory with the downgraded ΔP_x noted.

**Rationale**: The reviewer correctly identifies that "Counterfactual B displacement" is not the only positivity concern; the more fundamental issue is whether the counterfactual extrapolates beyond observed support. Quantitative pre-registration of the positivity criterion (10× threshold + 20% flagged-weight downgrade) prevents post-hoc rationalization of extrapolated counterfactual estimates.

**Implementation impact**: Stage 7 code module `code/stage7_counterfactual.py` must compute ρ_{c,x} for each (cell, counterfactual) pair before reporting ΔP_x. Total cost: ~1 hour.

### 4.6 m6 — SUTVA peer-effect language strengthened (c-Minor-6)

**Reviewer concern**: v2.0 Section 5.7.4 mentions "Counterfactual C displacement" as the SUTVA / consistency violation, but the broader concern in Counterfactual B is **classical interference** (Hudgens & Halloran 2008): if the intervention changes only the personality of individuals in Clusters 0/4/6, their workplace harassment behavior affects the observed outcomes of others through dyadic interaction. For a personality-shift counterfactual in a workplace context where harassment is dyadic, "personality change in person i has no effect on person j's measured harassment behavior" is dubious.

**Clarification (Level 1, applied to v2.0 master Section 5.7.4 + Discussion (eventual paper))**:

The Section 5.7.4 consistency assumption (item 3 of the four identifying assumptions) is rewritten with strengthened SUTVA language:

> "**Consistency / SUTVA (no-interference)**: The standard Stable Unit Treatment Value Assumption (SUTVA) requires that one individual's treatment status does not affect another individual's outcome. In a workplace harassment context, this assumption is **plausibly violated** because harassment is fundamentally a dyadic phenomenon: the perpetrator's behavior IS another individual's outcome. The counterfactuals modeled here therefore involve a degree of interference (Hudgens & Halloran 2008) that the simple do-operator framework does not formally capture.
>
> **Effect direction**: We anticipate that SUTVA violation is most relevant for Counterfactual B (targeted intervention). If positive peer effects exist (the intervention's effect on Cluster 0/4/6 individuals propagates to colleagues, reducing their harassment as well), the present analysis **underestimates** ΔP_B. If negative rebound effects exist (e.g., colleagues compensate for the reduced harassment by perpetrating themselves), the analysis **overestimates** ΔP_B.
>
> **Mitigation**: The 4-tier judgment hierarchy (Section 5.4) and the positivity downgrade rule (Section 5.7.3 + this clarification log m5) provide partial defense by preventing point-estimate-only inference from being treated as confirmatory. However, the SUTVA violation cannot be fully resolved within the present cross-sectional simulation framework. Future work with multilevel (workplace-clustered) data and explicit peer-effect modeling (e.g., Aronow & Samii 2017 partial interference; VanderWeele 2017) is required.
>

### 4.7 m7 — H7 intersection-union test specification (d-Minor-7)

**Reviewer concern**: v2.0 Section 6.3 states that H7 (ΔP_B > ΔP_A AND ΔP_B > ΔP_C) is "a single composite test, no further correction." This is correct under intersection-union framing, but v2.0 leaves the test statistic implicit. The pre-reg should specify which test: bootstrap-based simultaneous CIs with α/2 within the H7 family, or a likelihood-ratio-style joint test, etc.

**Clarification (Level 1, applied to v2.0 master Section 6.3)**:

H7 is operationalized as an **intersection-union test (IUT)** with bootstrap-based component CIs:

1. Compute the bootstrap distribution (B = 2,000) for each component contrast:
   - Δ_BA^{(b)} = ΔP_B^{(b)} − ΔP_A^{(b)} (positive value supports H7)
   - Δ_BC^{(b)} = ΔP_B^{(b)} − ΔP_C^{(b)} (positive value supports H7)

2. Compute one-sided lower 95% bootstrap BCa CIs for each component:
   - L_BA = 5th percentile of {Δ_BA^{(b)}}
   - L_BC = 5th percentile of {Δ_BC^{(b)}}

3. **H7 confirmation rule** (pre-registered):
   - **H7 CONFIRMED** if AND ONLY if: L_BA > 0 AND L_BC > 0 (both component lower bounds exceed zero).
   - **H7 PARTIAL** if: exactly one of {L_BA, L_BC} > 0 (one component supported, the other ambiguous).
   - **H7 REVERSAL** if: ΔP_B point estimate < ΔP_A point estimate OR ΔP_B point estimate < ΔP_C point estimate (Section 6.2 pre-registered failure case; publish per Section 7).
   - **H7 AMBIGUOUS** otherwise (point estimates support H7 but bootstrap CIs allow zero or reversal): reported as exploratory.

4. **No multiple-comparison correction** is applied within H7 because the IUT structure naturally controls the family-wise error rate at α = 0.05 by requiring BOTH components to meet the threshold simultaneously (Berger & Hsu 1996; Casella & Berger 2002 §8.2.6).

**Rationale**: IUT is the canonical test for "AND" composite hypotheses. The reviewer's concern was that v2.0 left the test form implicit; pre-registering IUT with one-sided bootstrap CIs at the 5th percentile makes the test fully reproducible.

**Implementation impact**: Stage 7 code module `code/stage7_counterfactual.py` must compute Δ_BA^{(b)} and Δ_BC^{(b)} from the same bootstrap iterations as ΔP_x^{(b)}, then derive L_BA and L_BC. Total cost: ~30 minutes.
