## 3. Major clarifications (M1–M4)

### 3.1 M1 — Bootstrap re-estimate (α̂, β̂) per iteration (a-Major-1)

**Reviewer concern**: v2.0 Section 5.2 estimates the Beta-Binomial hyperprior (α̂, β̂) by method-of-moments from the 14-cell point estimates {p̂_j}, then uses (α̂, β̂) as fixed prior for 28-cell shrinkage. Because each 14-cell estimate p̂_j is a deterministic function of the 28-cell counts whose posteriors are being constructed, the standard EB independence assumption (prior and data are independent draws from the hyper-population) is violated. Reported posterior CIs from Beta(α̂+X_k, β̂+N_k−X_k) are therefore narrower than honest credibility would warrant.

**Clarification (Level 1, applied to v2.0 master Section 5.2)**:

The bootstrap procedure used to construct CIs around the 28-cell EB-shrunken estimates re-estimates the hyperprior (α̂^(b), β̂^(b)) on each bootstrap iteration b ∈ {1..2,000}. The full per-iteration procedure is:

1. Resample N=354 with replacement, stratified by 14 cell (per Section 5.4 cell-stratified spec).
2. Re-classify each resampled individual to the nearest of 7 centroids (centroids fixed from N=13,668; per M3 below).
3. Compute the 14-cell observed propensities {p̂_j^(b)} for j = 1..14 on the resampled data.
4. **Re-estimate the hyperprior on iteration b**:
   - μ̂^(b) = mean(p̂_j^(b)), σ̂²^(b) = var(p̂_j^(b))
   - α̂^(b) = μ̂^(b) × [μ̂^(b)(1−μ̂^(b))/σ̂²^(b) − 1]
   - β̂^(b) = (1−μ̂^(b)) × [μ̂^(b)(1−μ̂^(b))/σ̂²^(b) − 1]
5. Apply the hyperprior (α̂^(b), β̂^(b)) to compute the 28-cell posterior on iteration b: E[p_k | X_k^(b), N_k^(b)] = (α̂^(b) + X_k^(b)) / (α̂^(b) + β̂^(b) + N_k^(b)).
6. Record the 28-cell posterior values for iteration b.

After all B = 2,000 iterations, 95% bootstrap CIs for each 28-cell posterior are constructed from the empirical distribution of {posterior^(b)}, with BCa correction (per Section 5.1 fallback chain if numerical instability detected).

**Rationale**: This adopts the reviewer's option (i) — "naive plug-in EB with bootstrap propagation" (Carlin & Louis 2000; Efron 2014) — as the operational choice. Option (ii) (full hierarchical Bayes via Stan as primary 28-cell estimator) remains available as the auxiliary triangulation per Section 5.2; the present clarification ensures the EB CIs reported in the main text honestly propagate hyperprior uncertainty.

**Implementation impact**: Stage 0 code module `code/stage0_eb_shrinkage.py` must re-estimate (α̂^(b), β̂^(b)) inside the bootstrap loop, not outside it. Computational cost is negligible (the MoM formula is O(K) for K=14 cells, dwarfed by the overall bootstrap cost).

### 3.2 M2 — Soft-assignment sensitivity sweep (a-Major-2)

**Reviewer concern**: v2.0 Stage 0 hard-assigns each N=354 individual to the nearest of 7 centroids by Euclidean distance, treating cluster membership as deterministic. Individuals near a centroid boundary (similar distances to ≥ 2 centroids) have their assignment fixed across all bootstrap iterations, systematically underestimating cell-level CIs by omitting cluster-assignment uncertainty.

**Clarification (Level 1, applied to v2.0 master Section 6.4 sensitivity sweep table)**:

A new soft-assignment sensitivity is added to the pre-registered sweep:

| Parameter | Main | Sweep range | Stage |
|---|---|---|---|
| **Cluster assignment scheme** | Hard nearest-neighbor (default, primary) | {Hard nearest-neighbor; Soft assignment with τ_softmax ∈ {0.5, 1.0, 2.0} × median nearest-neighbor distance} | 0 |

**Soft assignment specification**:

For each individual i and each cluster j ∈ {0..6}, define:

- d_ij = Euclidean distance from i's HEXACO 6-domain vector to centroid j
- d_i^(NN) = min_j d_ij (nearest-neighbor distance for individual i)
- median(d_i^(NN)) = median of d_i^(NN) across i = 1..354
- Temperature parameter τ = c × median(d_i^(NN)) for c ∈ {0.5, 1.0, 2.0}
- Soft-assignment weight: w_ij(τ) = exp(-d_ij / τ) / Σ_k exp(-d_ik / τ)

Under soft assignment, individual i contributes weight w_ij(τ) to cluster j's cell count rather than counting fully in one cluster. The 14-cell crosstab becomes:

X_c (soft, c = (j, gender)) = Σ_{i with gender = gender_c} w_ij(τ) × Y_i
N_c (soft, c) = Σ_{i with gender = gender_c} w_ij(τ)

where Y_i is the binary harassment outcome.

The full Stage 0 → Stage 1 → Stage 2 pipeline is run separately for each c ∈ {0.5, 1.0, 2.0} and reported alongside the hard-assignment main analysis.

**Robustness criterion**: if the national MAPE (Stage 2 primary) shifts by < 5 percentage points across c ∈ {0.5, 1.0, 2.0}, the hard nearest-neighbor assignment is reported as the primary specification with soft-assignment sensitivity supplied as supplementary. If MAPE shifts by ≥ 5 pp across c, the hard assignment is downgraded from "robust to assignment scheme" to "sensitive to assignment scheme", and the Discussion explicitly acknowledges that cluster-boundary individuals materially affect the inference.

**Implementation impact**: Stage 0 code module `code/stage0_type_assignment.py` must support both hard and soft assignment modes, parameterized by τ. Stage 0 → Stage 2 pipeline is run 4× total (1 hard + 3 soft τ values). The 4 runs share the same bootstrap seed (20260429) and centroid file.

**Clarification (Level 1, applied to v2.0 master Section 5.4)**:

### 3.4 M4 — Degenerate-cell handling (b-Major-4)

**Reviewer concern**: v2.0 Section 5.1 BCa fallback chain (BCa → BC → percentile) does not explicitly handle the fully degenerate case where a cell has all-zero (X_c = 0) or all-one (X_c = N_c) outcomes. The BCa acceleration parameter is undefined in this case (jackknife produces zero variance). In the 14-cell main analysis with min N = 10, this is improbable but possible. In the 28-cell EB sensitivity with cells of N = 1–3, it is virtually guaranteed for several cells. Silently producing NaN or [0, 0] CIs would be a Methods bug.

**Clarification (Level 1, applied to v2.0 master Sections 5.1 and 5.2)**:

A degeneracy rule is added at the top of the BCa fallback chain. The full per-cell CI selection logic, in priority order, is:

1. **Degeneracy check**:
   - If X_c = 0 (no perpetrators in cell): use the **exact Clopper-Pearson interval** for the binomial parameter with X = 0, N = N_c, confidence 95% [Clopper & Pearson 1934]. This yields CI = [0, 1 − (0.025)^(1/N_c)] for the upper bound only.
   - If X_c = N_c (all perpetrators in cell): use exact Clopper-Pearson with X = N_c, N = N_c. This yields CI = [(0.025)^(1/N_c), 1] for the lower bound only.
2. **BCa primary** (if not degenerate):
   - Compute jackknife acceleration parameter a per Efron (1987) Eq. 6.6 (per m4 below).
   - If |a| ≤ 10 and BCa computation succeeds without numerical error, use BCa interval.
3. **BC fallback** (if BCa fails):
   - If |a| > 10 OR BCa computation raises numerical error (e.g., division by zero at edge rates), use bias-corrected (BC) bootstrap.
4. **Percentile fallback** (if BC also fails):
   - Use simple percentile bootstrap (2.5th and 97.5th quantiles of the resample distribution).

For the 28-cell EB tier, degeneracy at the cell level is mitigated upstream by the EB shrinkage itself: the prior pulls the posterior off the {0, 1} boundary even when X_k = 0 or X_k = N_k. Therefore, the 28-cell tier uses the **EB posterior credible interval** (from Beta(α̂^(b) + X_k, β̂^(b) + N_k − X_k) quantiles, per M1) as the primary CI, with the bootstrap CI from the BCa/BC/percentile chain as auxiliary triangulation.

**Per-cell method recording**: every cell in the supplementary table records which CI method was actually used: {Clopper-Pearson, BCa, BC, percentile} for the 14-cell tier, and {EB posterior, bootstrap-BCa, bootstrap-BC, bootstrap-percentile} for the 28-cell tier. This enables readers to verify that the fallback chain's selections do not produce qualitatively different conclusions across cells.

**Rationale**: The reviewer correctly identified that the BCa fallback chain literally cannot run on degenerate cells (acceleration undefined). Clopper-Pearson is the exact-binomial fallback for the 14-cell case (X_c ∈ {0, N_c}); the EB posterior is the natural fallback for the 28-cell case where the prior provides regularization. Both are standard methods; no new statistical machinery is introduced.

**Implementation impact**: Stage 0 code module `code/utils_bootstrap.py` (or equivalent CI utility) must implement the 4-step priority chain. Standard libraries (`scipy.stats.binom.interval` for Clopper-Pearson; `scipy.stats.beta.interval` for the EB posterior) cover both fallbacks. Total additional implementation cost: ~2 hours.
