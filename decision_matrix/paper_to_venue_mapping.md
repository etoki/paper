## 推奨投稿割り当て案

### Plan A: バランス重視(現実シナリオ)

| 学会 | 投稿論文 | 理由 |

| 学会 | 投稿論文 | 理由 |

→ 結果: ECEL Proceedings (Scopus+WoS) + IEEE Xplore = SFC評価最強。ただし IEEE TALE virtual 可否未確認 = リスクあり

## 各論文の "学会用 conference paper 化" 必要作業

### P3 (meta-analysis) — ECEL用

- [x] 既に preprint manuscript あり(`metaanalysis/paper/manuscript_preprint.docx`)
- [ ] ECEL の 300-350語 abstract 作成
- [ ] ECEL の full paper format に合わせた整形(`metaanalysis/paper_v2/manuscript_journal_heliyon_v2.docx` から派生可能)
- [ ] preprint disclosure 文を cover letter に追加
- [ ] (任意) preprint migration: Research Square → OSF / arXiv

### P5 (simulation) — IEEE TALE用

- [x] preprint あり(OSF + SocArXiv)
- [ ] OSF / SocArXiv の正確な DOI を確認・記録
- [ ] IEEE 4-6ページ conference paper format で書き直し
- [ ] `simulation/paper_IEEE/` の素材を活用
- [ ] EDAS 投稿用 IEEE PDF eXpress 通過
- [ ] IEEE Copyright Form 準備

### P1 派生(Frontiers論文の追加分析) — ICEEL用

- [ ] Frontiers論文の差分を明確化(reused vs new)
- [ ] StudySapuri データ追加分析(`online_learning/0723/`, `1026/` 活用)
- [ ] ICEEL 8-10ページ format で執筆
- [ ] zmeeting 投稿システム対応
